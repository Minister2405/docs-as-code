# Core Concepts

A clear view of the main documentation sections and how they work together.

## What this guide covers

- The purpose of guides and reference pages
- How to organize content for readability
- How to keep docs easy to scan

## Documentation structure

### Guides
- Use for tutorials, workflows, and how-to content
- Keep each guide focused on one task

### Core concepts
- Explain key ideas users need to understand your docs
- Link to foundational pages and examples

### API reference
- Use for endpoint details and examples
- Keep it separate from how-to guides

<Tip>Short sections and clear headers make docs easier to browse.</Tip>

## Start here

- `/quickstart` — launch your docs quickly
- `/essentials/overview` — understand the docs structure
- `/api-reference/overview` — see how API docs are organized

<Warning>A consistent structure helps readers find information faster.</Warning>
